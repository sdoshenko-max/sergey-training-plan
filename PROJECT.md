# Проект: план тренировок Сергея

## Что это

Локальный и опубликованный статический сайт с планом тренировок на 3 месяца для обычного зала.

Главная задача сайта: открыть с компьютера или телефона и быстро понять, какая тренировка стоит на конкретной неделе и в конкретный день.

## Ссылки

- Публичный сайт: https://sdoshenko-max.github.io/sergey-training-plan/
- GitHub repo: https://github.com/sdoshenko-max/sergey-training-plan
- Локальная папка: `D:\Cloude CODE\training-plan-site`

## Текущий формат плана

- Период: 13 недель, с 22 июня 2026 по 18 сентября 2026.
- Частота: 3 тренировки в неделю.
- Дни: понедельник, среда, пятница.
- Формат недель: раскрывающиеся списки по неделям и дням.
- Формат упражнений: каждое упражнение внутри дня тоже раскрывается.
- Внутри упражнения обязательно есть:
  - инвентарь;
  - короткое описание техники.

## Сплит

Одна основная группа мышц тренируется один раз в неделю:

- Понедельник: грудь + трицепс.
- Среда: спина + бицепс.
- Пятница: плечи + пресс/корпус.

Ноги в план не добавлять. Кардио в план не добавлять.

## Ограничения по объему

- В каждой тренировке максимум 6 упражнений.
- Тренировка должна укладываться примерно в 60 минут.
- Не добавлять лишние добивки, если они не дают заметного качества.
- При сокращении сначала убирать дублирующие жимы, вторую изоляцию или низкоприоритетные добивочные упражнения.

## Важные требования

- Спина: не делать упражнения через боль.
- Для корпуса оставлять безопасные анти-скручивания и стабилизацию: `Dead bug`, `Pallof press`, `Suitcase carry`, планки.
- Нагрузка: почти все подходы делать с запасом 1-2 повтора.
- Прогрессия: если верхняя граница повторов сделана во всех подходах, можно добавить вес.
- Легкие недели: 6-я и 12-я недели.
- Белок как ориентир: 120-140 г/день. На сайте сейчас это намеренно не вынесено отдельным блоком, чтобы не было лишней воды.

## UI-правила

- Без маркетингового текста.
- Главный экран должен показывать только план: недели, дни, тренировки.
- Дни раскрываются по клику.
- Упражнения раскрываются по клику внутри открытого дня.
- Дизайн спокойный, читаемый, в светлом нейтральном стиле.
- Мобильная версия обязательна, без горизонтального скролла.

## Файлы

- `index.html` - минимальная HTML-оболочка.
- `styles.css` - стили сайта и адаптив.
- `app.js` - все данные плана, рендер недель/дней/упражнений и логика раскрытия.
- `test-site.mjs` - локальные проверки, не публикуется в GitHub Pages.
- `.nojekyll` - отключает Jekyll на GitHub Pages.
- `.gitignore` - исключает локальные служебные папки и тестовый файл из публикации.

## Как обновлять сайт

1. Изменить данные или интерфейс в локальной папке.
2. Проверить локально:

```powershell
node .\test-site.mjs
```

3. Проверить в браузере локальную или опубликованную версию.
4. Закоммитить изменения.
5. Запушить в `main`.
6. GitHub Pages сам обновит публичную ссылку.
